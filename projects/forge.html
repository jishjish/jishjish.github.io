PyForge

Python · CUDA · C++ · GPUs
GitHub ↗

A Python-to-CUDA compiler that dynamically detects your GPU at runtime via the CUDA driver API, queries hardware capabilities — thread dims, SM count, compute capability — and uses that information to emit optimized CUDA kernels as compiled .cu files.

Architecture

Built around a plugin-style device discovery system, Pydantic models for device state, and a clean separation between device detection, kernel generation, and compilation. The ctypes FFI layer bridges Python and the CUDA driver without requiring a full C extension.

What makes it interesting

Most Python GPU tooling abstracts away the hardware. Forge goes the other direction — it surfaces thread/block hierarchy, SM counts, and compute capability explicitly, then uses that information to parameterize kernel generation. The output is valid, compiled .cu code, not a wrapper.

Roadmap

The next step is a working matmul kernel end-to-end — two input arrays, a parameterized CUDA f-string, compiled via nvcc, result returned to Python. From there: tiled shared memory optimization, additional ops, and eventually a lightweight computation graph for kernel fusion.
