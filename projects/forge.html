Forge

Python · CUDA · Metal · C++ · GPUs
GitHub ↗

A runtime GPU compiler targeting portfolio analytics workloads. Forge detects your GPU at runtime, queries hardware capabilities (thread dims, SM count, compute capability) and uses that information to emit optimized kernels for operations like log returns and correlation matrices.

Why GPU for portfolio math

Operations like correlation matrices, efficient frontier computation, and rolling return windows scale with the number of assets and length of your time series. GPU acceleration lets you run these calculations across far larger universes and longer histories than traditional CPU-bound implementations typically allow. Forge generates compiled kernels parameterized to your actual hardware, not generic defaults.

Architecture

Built around a plugin-style device discovery system, Pydantic models for device state, and a clean separation between device detection, kernel generation, and compilation. The ctypes FFI layer bridges Python and driver without requiring a full C extension.

Target ops

log_returns
First kernel. End-to-end codegen target.
corr_matrix
N-asset pairwise correlation, tiled.
eff_frontier
Portfolio weight optimization across many combinations.

Roadmap

End-to-end log_returns kernel: two input arrays, a parameterized f-string template, compiled via nvcc, result returned to Python. From there: tiled shared memory optimization, correlation and frontier ops, and eventually a lightweight computation graph for kernel fusion across a full analytics pipeline.

Status

Early development. Testing on Metal now. Cloud GPU access needed for CUDA validation.
