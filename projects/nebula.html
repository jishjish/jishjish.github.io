Astrotek Nebula ↗

Astrotek Nebula serves as the backbone / orchestration platform for managing EVM based data. The worflow is sequential and builds on helpful open source projects for extracting data from the blockchain. The main goal of Nebula is to extract, process, classify, price, and tensorify erc20 transfers and logs.

The primary focus is ERC-20 transfers and event logs — extracting them from the chain, classifying transfer types, attaching pricing data, and producing tensor outputs suitable for downstream ML workloads.

Design

Built on top of proven open source tooling for chain data extraction, with custom processing layers on top. Each stage in the pipeline is sequential and composable — outputs from one stage feed directly into the next with no intermediate complexity.

  1. Erc20Data (Data) → src/evm/data.py

    Class to manage addition of block data and retrieval from corresponding parquet files. This can be thought of as an organizer class to get the data in line for future analysis. This is set to retrieve erc20 transfers and logs at the same time. For ex, requesting data for blocks 100 to 200 will get both transfers and logs via cryo.

  2. Erc20Ops (Ops) → src/evm/ops.py

    Handles parquet orchestration and consolidation of files, specifically managing / merging erc20 transfers and logs from their corresponding parquet repositories inside of the data folder.

  3. Erc20Classifier (Classifier) → src/evm/classifier.py

    Loads transfer and log parquets into Polars lazyframes. Zip and iterate through the file paths and lazyframes to match exchange / event (match_exchange_and_event) and token metadata (match_token_metadata), finally, write output to parquet with prefix "matched".

Output

Tensorified ERC-20 transfers and logs that can feed directly into models for anomaly detection, wallet classification, and on-chain behavioral analysis.
